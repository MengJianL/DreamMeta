# Reports

Project test reflection reports — structured post-mortem documents that feed back into Meta-Department governance iterations.

## Naming Convention

```
RPT-{序号}-{项目代号}-{类型}-{日期YYYYMMDD}.md
```

| Field | Description | Example |
|-------|-------------|---------|
| 序号 | Auto-increment, 3 digits | 001, 002, ... |
| 项目代号 | Short project codename | cocoon-portfolio |
| 类型 | reflection / postmortem / audit | reflection |
| 日期 | Report creation date | 20260409 |

## Usage

These reports serve as **input to `/meta` self-iteration cycles** — the Meta-Department reads them and evolves its own atom definitions and governance mechanisms accordingly.
